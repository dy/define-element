Intro

Web-components by example

Define custom elements in HTML. No JS required.

index.html 
<define-element>
  <x-counter count:number="0">
    <template>
      <button part="btn">
        Clicks: <span part="n">0</span>
      </button>
    </template>
    <script>
      this.part.btn.onclick = () => {
        this.count++
        this.part.n.textContent = this.count
      }
    </script>
  </x-counter>
</define-element>

<!-- use it -->
<x-counter></x-counter>
<x-counter count="10"></x-counter>

Features

No API

The API is markup

Props

Declare with optional types: count:number="0". Props reflect to attributes and back.

Styles

Light DOM scopes via CSS nesting, :host works in both. Shadow DOM uses adoptedStyleSheets.

Shadow DOM & Slots

shadowrootmode for encapsulation with native slots for content projection.

Parts

Elements with part="name" are collected into this.part to avoid queries.

Lifecycle

onconnected, ondisconnected, onadopted, onattributechanged. Script runs once per instance.

No eval

Scripts run via element injection, not eval or new Function. CSP-compatible.

Lifecycle

Connect, disconnect, update

Script runs once on first connect. Hooks fire on connect, disconnect, attribute change.

clock.html 
<define-element>
  <x-clock>
    <template>
      <time part="t"></time>
    </template>
    <script>
      let id
      const tick = () =>
        this.part.t.textContent =
          new Date().toLocaleTimeString()
      tick()
      this.onconnected = () =>
        id = setInterval(tick, 1000)
      this.ondisconnected = () =>
        clearInterval(id)
    </script>
    <style>
      :host { font-family: monospace }
    </style>
  </x-clock>
</define-element>

    Why
    
Why this exists

    
The W3C Declarative Custom Elements proposal has stalled for years. JS-side solutions require build tools and class boilerplate. The declarative CE space is a graveyard.

    
The gap: no lightweight way to define a custom element as HTML. Paste a <define-element> block into any page, CMS, or markdown and it works.

    
This ~270-line polyfill is evidence that the proposal is implementable and useful. Ship it natively.
