See It Work

Define once, use anywhere

Write the component as HTML. Use it as HTML. That's it.

index.html 
<define-element>
  <x-counter count:number="0">
    <template>
      <button part="btn">
        Clicks: <span part="n">0</span>
      </button>
    </template>
    <script>
      this.part.btn.onclick = () => {
        this.count++
        this.part.n.textContent = this.count
      }
    </script>
  </x-counter>
</define-element>

<!-- use it -->
<x-counter></x-counter>
<x-counter count="10"></x-counter>

Features

Everything a component needs

Standard web component capabilities, declared in HTML.

Typed Props

Declare with types: count:number="0". Properties reflect to attributes and back.

Scoped Styles

Shadow DOM uses adoptedStyleSheets. Light DOM scopes via CSS nesting. :host just works.

Shadow DOM & Slots

Add shadowrootmode for encapsulation. Native slots for content projection.

Parts

Elements with part="name" are collected into this.part. DOM refs without queries.

Lifecycle

onconnected, ondisconnected, onadopted, onattributechanged. Script runs once per instance.

No eval

Scripts run via element injection, not eval or new Function. CSP-compatible.

Lifecycle

Connect, disconnect, react

Script runs once on first connect. Lifecycle hooks handle the rest.

clock.html 
<define-element>
  <x-clock>
    <template>
      <time part="t"></time>
    </template>
    <script>
      let id
      const tick = () =>
        this.part.t.textContent =
          new Date().toLocaleTimeString()
      tick()
      this.onconnected = () =>
        id = setInterval(tick, 1000)
      this.ondisconnected = () =>
        clearInterval(id)
    </script>
    <style>
      :host { font-family: monospace }
    </style>
  </x-clock>
</define-element>

    Why
    
The missing piece

    
The W3C Declarative Custom Elements proposal has stalled for years. JS-side solutions require build tools and class boilerplate. The declarative CE space is a graveyard.

    
The gap: no lightweight way to define a custom element as HTML. Paste a <define-element> block into any page, CMS, or markdown and it works.

    
This ~300-line polyfill is evidence that the proposal is implementable and useful. Ship it natively.
